<entry [Wed Apr 21 10:48:45 EDT 2010] GET REFSEQ IDS AND GENE NAMES FOR knownGenes>




dario:

Let me explain better cause this might be useful for all of us (or maybe I was the only one ignorant about that), even when performing more Cufflinks analysis with different .gtf for different classes.

In order to have a .gtf as Stuart showed us we access the Table page of Genome Browser

http://genome.ucsc.edu/cgi-bin/hgTables?org=Mouse&db=mm9&hgsid=156394747&hgta_doMainPage=1

In here we choose the clade, genome, assembly and so on. 
Settings Stuart for this analysis (I suppose...) are:

group: Genes and Gene Prediction Tracks
track: USCS Genes
table: knownGene
Output format: all fields from selected table

(I just name the fields that are of any interest to us)

If on the last field you choose

Output format: selected fields from primary and related tables

And then you click on “get output”
It will open a new page where you can choose what cloumns/fields you will get in the output from your table of interest (in this case the knownGene table). Moreover you will find the list of all the other possible tables existing. If you check one of it and click on “Allow selection from Checked Tables” it will open the fields list also from the new table you checked.

Remember that in order to work the new tables must have at list one field in common with the principal one.

Actually I found a comment on a bioinfo website suggesting to do this, and they were suggesting to cross the knownGene table with the refLink table to get useful IDs.

Actually that’s what I did. Instead of running Cufflinks again, we might want to just cross the files we have now (.expr and .gtf) with the Ref tab delimited text I created. Still first we need the results for the alignement against the rat genome, I think.

Whoever wants the Ref file just tell me.

Dario
